# NanoDAW Orbit Notes

Short development dispatches live here. They complement the queue and loop
reports; they do not replace product or architecture contracts.

Use this shape:

```md
# YYYY-MM-DD - <orbit title>

## Ce qu'on a construit

## Pourquoi ca compte

## Ce qui reste bancal

## Prochain orbit
```

Rules:

- distinguish deterministic tests, browser evidence, and listening evidence;
- never present a disposable prototype as a committed product direction;
- name absent external runtimes explicitly;
- keep each note concrete and short;
- link the corresponding loop report and decision record.
