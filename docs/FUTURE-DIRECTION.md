# Future Direction: Arrangement Assistance

Beat Twin should stay conservative as it grows. The useful next product direction is not autonomous composition; it is safer arrangement assistance that explains what it would do before any write tool is enabled.

## Near-Term Shape

- Inspect the visible Bitwig session.
- Summarize available tracks, scenes, transport state, selected track, and selected device.
- Produce an arrangement plan with sections, intent, risks, missing data, and required write policies.
- Keep the first implementation plan-only.

## Write-Side Preconditions

Arrangement execution should wait until Beat Twin has:

- stronger preview output;
- explicit policy gates per write class;
- clear rollback guidance;
- manual live-smoke coverage in disposable projects;
- better visibility into clip names, lengths, and scene content.

## Non-Goals For Now

- No hidden autonomous DAW mutation.
- No claims of production-ready arrangement generation.
- No broad write mode as the default user experience.

The project should earn trust through narrow, inspectable workflows before it expands the creative surface.
